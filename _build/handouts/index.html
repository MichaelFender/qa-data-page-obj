Data Driven Testing & Page Objects

Introduction

In this exercise, you’ll be working with the Employee Manager app and a To Do app to explore data driven testing and using page objects.

Employee Manager

Setup

  1. Download the materials for this exercise from Frodo.

  2. In your terminal, navigate to the folder you just downloaded, cd into the employeeManager folder, and run npm install.

  3. The live site can be viewed here.

Instructions

  1. Head to Employee Manager and explore it a bit.

  2. Look for a test that can be repeated multiple times.

  3. Identify each step you’d take for this test in detail.

  4. Identify the data needed for each step.

  5. Create multiple sets of data (inputs and expected results) for the test you identified.

  6. In employee.test.js, define a class to hold each data set, and build out an array of each set.

  7. In employee.test.js, write the code for the test that uses your data.

To Do

Setup

  1. In your terminal, navigate to the folder named toDo from the Frodo materials for this lab exercise.

  2. Run npm install.

  3. The live site can be seen here.

Instructions

  1. Review the tests provided for you.

  2. In firstTest.test.js, create locators for todoInput, todos, todoLabel, todoComplete, and clearCompletedButton.

    For todoLabel and todoComplete, you can look in the test and see that the test is looking for these elements FROM the todo element, so we only need a unique selector from the li.todo element.

  3. Head to todoTests.test.js and determine what functionality you will test – you can use the outline provided as a jumping off point, but can add what you’d like. And make sure that you’re not just repeating firstTest.test.js.

  4. Outline the steps you’ll need to follow to execute these tests.

  5. Make a list of all the elements you’ll need locators for to be able to execute these tests.

  6. Place them all as properties of the TodoPage class.

  7. Create your tests. (You don’t need any set number of tests here; so long as you have a page object and some testing passing, you’re good to go.)

  8. Run your tests with npm run test

When you’re done, push everything to GitHub.
